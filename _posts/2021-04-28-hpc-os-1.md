## Cahpter1. Computer system overview  


#### 컴퓨터 하드웨어
* 프로세서(Processor)
  + CPU, GPU(그래픽카드) 
* 메모리(Memory)
  + Main memory(주기억장치), Secondary memory(보조 기억 장치)
* 주변장치
  *  키보드, 마우스, 모니터 등등



먼저 프로세서(Processor)에 대해 알아보자
### Processor는 중앙처리장치로 연산을 수행하고 cpu의 모든 장치의 동작을 제어한다.
![](https://siiiido.github.io/assets/images/ch1-1.png)


즉 **Processor는 CPU라고 보면 좋다.**

* 프로세서(Processor)와 프로세스(Process)는 다르다  
* 프로세서(Processor)은 계산(연산)과 제어를 한다.  




메모리(Memory)에 대해 알아보자
### 메모리는 데이터를 저장(기억)하는 장치이다. 
![](https://siiiido.github.io/assets/images/ch1-2.png)


이 그림은 메모리의 종류를 나타낸 것이다.
* 위로 갈 수록
  * Expensive
  * Fast
  * Small
* 아래로 갈 수록
  * Chip
  * Slow
  * Big
 
 *레지스터, 캐시, 메인 메모리*는 프로세서(Processor)가 프로그램과 데이터에 직접 접근할 수 있지만 *보조기억장치*는 프로그램 데이터를 **메인 메모리**에 옮겨야 실행이 가능하다. 

메모리의 종류를 알아보자

1. ### 레지스터(Register)

      레지스터는 메모리니깐 당연히 프로세서가 사용할 데이터를 저장하고 컴퓨터에서 가장 빠른 메모리이다.
레지스터는 여러 종류가 있는데
   * 용도에 따른 분류
     * 전용 레지스터
     * 범용 레지스터  <범용 = 일반적인>
   * 사용자가 정보 변경 가능 여부에 따른 분류
     * 사용자 가시 레지스터 <가시 = 변경 가능>
     * 사용자 불가시 레지스터 <불가시 = 변경 불가능>
   * 저장하는 정보의 종류에 따른 분류
     * 데이터 레지스터
     * 주소 레지스터
     * 상태 레지스터
  
    대표적인 예시들을 보고 가겠다.
     * 프로그램 카운터(PC) : *사용자 불가시 레지스터*이면서 *전용 레지스터*이다. *다음에 실행할 명령어의 주소*를 보관한다.
     * 명령어 레지스터(IR) : *현재 실행하는 명령어*를 보관한다.
     * 누산기(ACC) : 데이터를 *일시적*으로 저장한다.

2. ### 캐시(Cache)
      캐시는 일반적으로 하드웨어가 관리하고 **메인 메모리의 입출력 병목현상을 해소** 한다.

      *(병목현상은 "방해"라고 봐도 좋다)*
    ![](https://siiiido.github.io/assets/images/ch1-3.png)

    위 그림을 보면 *빨간색인 CPU의 속도*와 *주황색인 Main memory 속도*의 Gap을 볼 수 있다. 이 Gap을 줄이기 위해 캐시를 사용한다. (밑에서 말하겠지만 주기억장치 역시 CPU와 Disk 속도의 Gap을 해결하기 위해 사용된다.)

    그리고 캐시는 
    * 캐시 히트(Cache hit)
    * 캐시 미스(Cache miss)
    ![](https://siiiido.github.io/assets/images/ch1-4.png)

    위 그림을 보면 프로세서(Processor)에서 캐시로 가서 필요한 Data가 있으면 캐시에서 바로 프로세서로 보내고 *<캐시 히트>* 필요한 Data가 없으면 *<캐시 미스>* 메인 메모리까지 가서 가져와야 한다. 

    즉 **속도**의 차이가 있다.

    그리고 캐시를 공부하면서 **지역성(Locality)**도 같이 공부하면 좋은데
    지역성은
    * CPU가 기억장치의 특정 부분에 위치한 Data나 program 코드를 집중적으로 접근하는 현상
    * 지역성의 특성을 이용하여 고비용/고속 저장 장치의 효율적 사용 및 속도를 향상 시킬 수 있다.

    지역성의 종류는
    * 공간적 지역성(Spatial locality)
      * 참조한 주소와 인접한 주소를 참조하는 특성
        * 배열과 같이 순차적 프로그램 수행
    * 시간적 지역성(Temporal locality)
      * 한 번 참조한 주소를 곧 다시 참조하는 특성
        * for문 등의 순환문

      **지역성은 캐시 적중률(cache hit ratio)와 밀접하다**

    ![](https://siiiido.github.io/assets/images/ch1-5.png)
    위 그림으로 지역성의 예를 볼 수 있다.  
    A와 B의 비교를 해보자 누가 더 빠를까?  **A가 더빠르다.**       
    WHY? 처음에 메인 메모리에서 캐시로 A[0][0~15]를 올린다. *<여기서 A는 A,B 비교의 A가 아니라 배열의 A이다. 헷갈릴 수도 있으니 이제부터 배열 A를 A[]라고 표현하겠다.>*
    A는 A[0][0], A[0][1], A[0][2]..A[0][15]로 캐시에 올라와 있어 빨리 가져가는데 <*캐시 히트*> B는 A[0][0], A[1][0], A[2][0]...A[15][0]로 계속 <*캐시 미스*>가 떠서 메인 메모리에서 다시 가져와야 한다. 즉 A와 B를 비교하면 A가 훨씬 더 빠르다.


3. ### 주기억장치(Main memory)  
      DRAM을 주로 사용하고 용령이 크고 가격이 저렴하다. 그리고 **디스크 입출력 병목 현상(I/O bottlenect)를 해소한다.**


    ![](https://siiiido.github.io/assets/images/ch1-6.png)
    위 그림처럼 캐시와 비슷하게 메인 메모리 역시 CPU와 Disk의 속도 Gap을 맞출력고 중간다리 같은 역할을 한다.


4. ### 보조기억 장치(Secondary memory, Auxiliary memory) 
   보조기억 장치는 **프로세서가 직접 접근 할 수 없다(주변장치)** 즉 주기억장치를 거쳐서 접근해야 한다.  
   용량이 크고 가격이 저렴하다. 하드디스크나 USB 등이 있다.


오늘은 이렇게 컴퓨터 하드웨어에 대해 알아봤다. 

핵심 키워드는
* 프로세서(Processor)
  * 프로세서는 연산 + 제어
  * 프로세서(Processor)은 프로세스(Process)와 다르다
* 메모리
  * 레지스터
    * Expensive, Fast, Small
  * 캐시
    * 캐시 히트
    * 캐시 미스
  * 메인 메모리
    * 디스크 입출력 병목 현상 해소
  * 보조기억장치
    * 주변장치로 프로세서가 직접 접근할 수 없음
    * Chip, Slow, Big
* 지역성
  * 공간적 지역성
  * 시간적 지역성
